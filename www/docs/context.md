# UCP Admin Dashboard — Domain Language

> Shared vocabulary for the AI agent and the team.
> One term = one definition. Update whenever a new concept gets a name.

---

## Core Concepts

| Term | Definition |
|------|-----------|
| **Dashboard** | The admin web UI for monitoring and managing UCP Server; embedded SPA (React) served from Go binary |
| **Session** | A user's authenticated connection; signed by Better Auth, stored in SQLite, time-limited (24h default) |
| **Role** | User permission level; either `admin` (full access) or `user` (read-only, limited actions) |
| **Envelope** | A UCP message wrapper (unencrypted metadata + encrypted payload); stored in UCP Server's Postgres database |
| **Thread** | A conversation group identified by `thread_id`; envelopes belong to exactly one thread |
| **Identity** | A UCP address (`localpart@domain`) representing a user or server; anchored in DNS via Ed25519 keypair |
| **Signing Key** | An Ed25519 key used to cryptographically sign envelopes; rotates every 60 days with 48-hour grace period |
| **Federation** | Server-to-server communication; messages may be routed to remote UCP servers by domain |
| **Bridge** | Legacy email integration; incoming SMTP/IMAP bridged to UCP (attestation marks these as non-E2E) |
| **MLS Group** | A Messaging Layer Security group (RFC 9420) for end-to-end encryption; one per thread |
| **Well-Known Endpoint** | DNS or HTTP route (e.g., `/.well-known/ucp/identity`) for bootstrapping federation and identity discovery |

## Domain Objects

| Term | Definition |
|------|-----------|
| **User (dashboard)** | A person with access to the admin dashboard; has email, password (Better Auth), role, preferences |
| **User (UCP)** | A person using the UCP protocol; identified by address (e.g., `alice@example.com`); has keypairs and signing key |
| **Message** | A UCP message envelope with `from`, `to`, `cc`, `thread_id`, body (encrypted), attachments |
| **Attachment** | A file uploaded with a message; encrypted separately, identified by SHA256 hash |
| **Server** | This UCP Server instance (or a remote UCP server); identified by domain and Ed25519 public key |
| **Stats** | Aggregate metrics: message count, active connections, uptime, federation peer status |
| **Audit Log Entry** | A record of an admin action (login, user create, message delete, config change); immutable, timestamped |
| **Config** | Server settings visible to admins (read-only mirror of Go server config in dashboard DB) |
| **Peer** | A remote UCP Server we federate with; identified by domain |

## States

| Term | Definition |
|------|-----------|
| **Session Valid** | Session token verified, signature valid, not expired, user found in DB |
| **Session Expired** | Session token past `expiresAt` timestamp; user must re-authenticate |
| **Session Revoked** | User logged out; session marked deleted; new requests rejected |
| **Message Pending** | Envelope received but not yet confirmed by recipient's server (federation retry queue) |
| **Message Delivered** | Recipient's server ACK'd delivery; message durably stored there |
| **Message Failed** | Delivery failed after 48-hour retry window; bounce generated or logged |
| **Identity Rotating** | Signing key in grace period (7 days before expiry); old key still valid for verification |
| **Identity Expired** | Signing key past grace period; envelope signatures from old key rejected |
| **Peer Connected** | Federation connection established and healthy (ping/pong active) |
| **Peer Disconnected** | Federation connection lost; messages queued for retry |

## Abbreviations

| Abbrev | Expands to |
|--------|-----------|
| **UCP** | Unified Communications Protocol |
| **E2E** | End-to-end (encryption) |
| **MLS** | Messaging Layer Security (RFC 9420) |
| **RFC** | Request for Comments (IETF standard) |
| **SMTP** | Simple Mail Transfer Protocol (legacy email inbound) |
| **IMAP** | Internet Message Access Protocol (legacy email outbound bridge) |
| **DNS** | Domain Name System |
| **TLS** | Transport Layer Security (HTTPS) |
| **HTTP** | Hypertext Transfer Protocol |
| **API** | Application Programming Interface |
| **DB** | Database (SQLite, in this context) |
| **ORM** | Object-Relational Mapping (Drizzle, in this context) |
| **TS** | TypeScript |
| **SPA** | Single-Page Application (React) |
| **OpenAPI** | Machine-readable API specification (auto-generated by Hono) |
| **Middleware** | HTTP request interceptor (auth verification, logging, etc.) |
| **Handler** | HTTP route handler function (receives request, returns response) |
| **Route** | HTTP endpoint path + method (e.g., `GET /api/stats`) |
| **Schema** | Drizzle table definition or Zod validation schema |
| **Migration** | Database schema version change (applied via `drizzle-kit push`) |
| **TTL** | Time-to-live (expiry duration) |
| **ACK** | Acknowledgment (server confirmed receipt) |

## Status Codes (API)

| Code | Meaning | Context |
|------|---------|---------|
| **200** | OK | Successful request, response body included |
| **201** | Created | Resource created (e.g., new user) |
| **204** | No Content | Successful, no response body |
| **400** | Bad Request | Invalid input (Zod validation failed) |
| **401** | Unauthorized | Missing/invalid session token |
| **403** | Forbidden | Authenticated but lacking permission (e.g., not admin) |
| **404** | Not Found | Resource doesn't exist |
| **500** | Internal Server Error | Unhandled exception; check server logs |
